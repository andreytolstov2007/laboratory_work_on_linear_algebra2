1. Что понадобилось изучить для написания работы:
  1) random.seed(): Если запустить такой алгоритм с одним и тем же начальным значением (seed), он всегда будет производить абсолютно одинаковую последовательность "случайных" чисел.

  2) random.gauss(a, b): Генерирует одно случайное число, подчиняющееся нормальному (гауссовскому) распределению.
  3) random.sample(a, b): Возвращает список из b уникальных элементов, выбранных из последовательности a.
  4) Gradient Descent with Momentum: Идея момента заключается в добавлении инерции к движению. Скорость: Vnew=β⋅Vold−α⋅∇L, где β (momentum) — коэффициент сохранения импульса. α (learning rate) — размер шага. ∇L — градиент функции потерь. Вес: Wnew = Wold + Vnew
​  5) лучше изучена библиотека для построения графиков.

2. Функции перцептрона:
  1) __init__(self, input_dim, init_method='small_random'): Конструктор класса. Принимает размерность входных данных (input_dim) и метод инициализации весов (init_method). Вызывает метод Initialize_weights().
  2) Initialize_weights(self): Инициализирует вектор весов W и смещение b. Поддерживает три метода: 'zero','small_random','large_random'.
  3) Прямое распространение (Forward Pass):
    1. Сигмоида: Вычисляет значение сигмоидной функции активации.
    2. Forward_single(self, x): выполняет прямой проход для одного примера x. Вычисляет взвешенную сумму z = (W^T)*x + b и пропускает её через сигмоиду, получая вероятность принадлежности к классу 1.
    3. Forward(self, X): применяет Forward_single ко всему набору данных X, возвращая список предсказанных вероятностей.
  4) Обучение (Training):
    1. Compute_loss(self, y_true, y_pred): Вычисляет бинарную кросс-энтропию (функцию потерь), которая измеряет расхождение между истинными метками y_true и предсказанными вероятностями y_pred.
    2. Compute_gradients(self, X_batch, y_batch, y_pred_batch): вычисляет градиенты функции потерь по весам dW и смещению db для заданного батча данных. Используется формула ошибки (y_pred - y_true) для логистической регрессии.
    3. Update_with_momentum(...)
    4. Fit(self, ...): организует цикл по эпохам, внутри которого перемешивает данные, формирует батчи, выполняет прямой и обратный проходы, обновляет веса и вычисляет потери на обучающей и валидационной выборках после каждой эпохи.
  5) Предсказание и оценка: 
    1. Predict_single(self, x): преобразует выход сигмоиды (вероятность) в класс (0 или 1) по порогу 0.5.
    2. Predict(self, X): возвращает предсказанные классы для всего набора данных X.


3. Выводы по результатам работы: 
  1) Скорость обучения (Learning Rate). η=0.01: Оптимальная скорость. Модель стабильно сходится, достигая низких значений функции потерь.
  2) Размер батча (Batch Size):
    1. Batch Size = 1 (SGD): Очень шумный, но быстрый путь к минимуму. После каждой итерации веса сильно меняются, что приводит к "дерганой" кривой потерь. Однако это позволяет быстро выйти из плохих локальных минимумов.
    2. Batch Size = 16/32/64 (Mini-batch): Компромисс между скоростью и стабильностью. Шум меньше, чем у SGD, но обновления происходят достаточно часто. Этот режим является стандартом.
    3. Batch Size = 256 (ближе к пакетному GD): Плавное, стабильное снижение функции потерь. Требует больше памяти и медленнее реагирует на изменения в данных.
  3) Инициализация весов:
    1. zero: Модель не смогла обучиться. Поскольку все веса и их градиенты изначально равны нулю.
    2. mall_random: Модель успешно обучилась, показав хорошие результаты как на обучающей, так и на тестовой выборке.
    3. large_random: Начальное значение функции потерь было очень высоким из-за больших начальных активаций нейронов. Обучение шло медленнее, чем у mall_random, так как требовалось сначала уменьшить веса до приемлемого уровня.
4. Градиентный спуск с моментом (Momentum):
  1) β=0.0 (обычный SGD): Базовый уровень. Сходимость идет, но может быть медленной, особенно в узких "оврагах" поверхности потерь.
  2) β=0.5: Заметно ускорение сходимости по сравнению с обычным SGD.
  3) β=0.9: Значительное ускорение. Импульс эффективно "проталкивает" алгоритм через пологие участки и осцилляции, приводя к быстрой сходимости. При данном β точность на тесте примерно равна 0.94 (наилучшая точность среди данных β).
  4) β=0.99: Наблюдается нестабильность. Из-за высокого импульса алгоритм "проскакивает" минимум и начинает осциллировать вокруг него с большой амплитудой, что может привести к расхождению.
